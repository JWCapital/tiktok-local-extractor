<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TikTok platform → local pipeline | Untrusted remote video bytes and metadata (description, caption, link_in_bio) cross into the local filesystem via yt-dlp/extract.py. |
| Browser cookie store / cookie file → yt-dlp | Authenticated session credentials are handed to yt-dlp to download favorited/private content. |
| _staging/tiktok → Inbox-Raw/tiktok | Finalize promotes content into the Hive-polled inbox lane (downstream auto-ingestion boundary). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | batch_extract.sh inline `python -c` link extractor (info_json path interpolated into the script string) | low | accept | Path is a fixed glob under the controlled `_staging/tiktok` tree; the snippet only runs read-only regex over description text and prints results — no shell-eval of remote content. |
| T-02-02 | Information disclosure | Brave browser cookies / `cookies/tiktok_cookies.txt` used for authenticated TikTok access | medium | mitigate | Cookies are passed only as yt-dlp args, never echoed to batch_extract.log/stdout; the cookie file stays under `_assets/tiktok/cookies/` (outside the privacy-restricted `private/`, `client-data/` paths) and is not committed. |
| T-02-03 | Denial of service | Sequential requests across 186 URLs may trip TikTok rate-limit / IP block | low | accept | Script sleeps 1s between requests; failures are recorded to failed_ids.txt and skipped (not retried) per D-02. |
| T-02-04 | Tampering | Untrusted downloaded media/metadata written to the filesystem and later promoted to the Hive lane | low | mitigate | Content lands in dedicated `_assets`/`_staging` trees; `--finalize-all` validates required contract fields and enforces `routing_zone: work` before moving into `Inbox-Raw/tiktok/`. |

No package installs in this plan — the `.venv` is already provisioned; supply-chain (T-02-SC) gate is N/A.
</threat_model>
